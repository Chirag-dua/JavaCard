final int com.gotrust.sesdapi.SESDAPI.SDSCCreateIoFile ( String  strIoFileDirectory)

Create an I/O file under the specified directory.

The function blocks the caller until the file creation process completes.

Parameters
[in]strIoFileDirectoryThe directory path where the I/O file should be created. Refer to SDSCListDevs() for detail explanation about the path.
Returns
Returns 0 on success; returns the extra space (in MB) required if the device is out of space.

final Vector<String> com.gotrust.sesdapi.SESDAPI.SDSCListDevs ( String  strIoFilePath)

Get the list of all the devices according to SDSC_DEVICE_PATH_NAME.

If SDSC_DEVICE_PATH_NAME is not assigned, it will search available devices by checking all mounted devices.

Parameters
strIoFilePathSupplies the I/O file path (including the file name) located on the secure microSD. For Android 4.4.2 (Kitkat), the I/O file path must be under the application directory located in secure microSD, because that's the only accessible directory on microSD. The application own directory is named by its package name and can be queried by using ContextWrapper.getExternalFilesDirs(). For Example, if an application's package name is com.gotrust, its own directory is "<microSD root>/Android/data/com.gotrust".
Returns
Returns the list of all found devices.
Exceptions
SDSCExceptionwill be thrown if any error occurs.
