DWORD __stdcall SDSCGetFirmwareVer ( IN HANDLE  hDevice,
OUT BYTE *  pbFirmwareVer,
IN OUT DWORD *  pulFirmwareVerLen 
)

Get the firmware version of the device.

Parameters
hDeviceSupplies the device handle obtained by calling SDSCConnectDev().
pbFirmwareVerReceives the firmware version of the device. The buffer size must not be smaller than SDSC_FIRMWARE_VER_LEN.
pulFirmwareVerLenSupplies the length of the pbFirmwareVer buffer in bytes, and receives the actual length of the received firmware version.
Returns
Returns 0 on success; otherwise, returns an error code.

DWORD __stdcall SDSCGetDeviceInfo ( IN HANDLE  hDevice,
OUT BYTE *  pbDeviceInfo,
IN OUT DWORD *  pulDeviceInfoLen 
)

Get the secure microSD device info. The content of the returned device info depends on product type. For the device of the type SDSC_PRODUCT_TYPE_NS, the device info is its flash ID. For the device of the type SDSC_PRODUCT_TYPE_CUP, the device info is the response of the SCIF_INFO command.

Parameters
hDeviceSupplies the device handle obtained by calling SDSCConnectDev().
pbDeviceInfoReceives the device info. The buffer must not be smaller than SDSC_DEVICE_INFO_LEN.
pulDeviceInfoLenSupplies the length of the pbDeviceInfo buffer in bytes, and receives the actual length of the received device info.
Returns
Returns 0 on success; otherwise, returns an error code.
See also
SDSCErr.h
DWORD __stdcall SDSCResetCard ( IN HANDLE  hDevice,
OUT BYTE *  pbAtr,
IN OUT DWORD *  pulAtrLen 
)

Reset smart card. If the smart card has not been reset, this library will do SDSCResetCard() automatically when the calling application firstly calls SDSCConnectDev().

Parameters
hDeviceSupplies the device handle obtained by calling SDSCConnectDev().
pbAtrReceives the ATR if resetting smart card successfully. The buffer must be big enough.
pulAtrLenSupplies the length of the pbAtr buffer in bytes, and receives the actual length of the received ATR.
Note
If the calling application needn't get the ATR, the parameters pbAtr and pulAtrLen can be NULL.
Returns
Returns 0 on success; otherwise, returns an error code.

DWORD __stdcall SDSCTransmit ( IN HANDLE  hDevice,
IN BYTE *  pbCommand,
IN DWORD  ulCommandLen,
IN DWORD  ulTimeOutMode,
OUT BYTE *  pbOutData,
IN OUT DWORD *  pulOutDataLen,
OUT DWORD *  pulCosState 
)

Send an APDU command to the device, and expect to receive data back from the device.

Parameters
hDeviceSupplies the device handle obtained by calling SDSCConnectDev().
pbCommandSupplies the APDU command to be sent to the device.
ulCommandLenSupplies the length (in bytes) of the pbCommand parameter.
ulTimeOutModeSpecifies the timeout to wait for a response. The timeout can set to one of the following values:
  • SDSC_DEV_DEFAULT_TIME_OUT
  • SDSC_DEV_MAX_TIME_OUT
  • SDSC_DEV_CUSTOM_FLAG_TIME_OUT
pbOutDataReceives the APDU response from the device (not including two-byte APDU status word).
pulOutDataLenSupplies the length of the pbOutData parameter (in bytes) and receives the actual number of bytes received from the device.
pulCosStateReceives the APDU status word.
Returns
Returns 0 on success; otherwise, returns an error code.
See also
SDSCErr.h
