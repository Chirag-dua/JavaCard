unsigned long SDSCSetProductType ( int  ulProductType)

Specify the product type. If users do not specify the product type, the default type is SDSC_PRODUCT_TYPE_AUTO_DETECT. In this case, the library will try to detect the product type in SDSCListDevs(). SDSCListDevs() can have better performance if the product type is specified explicitly.

Parameters
[in]ulProductTypeThe product type. It can be one of the following values:
  • SDSC_PRODUCT_TYPE_AUTO_DETECT
  • SDSC_PRODUCT_TYPE_NS
  • SDSC_PRODUCT_TYPE_CUP
Returns
Returns 0 on success; otherwise, returns an error code.
unsigned long SDSCInitParams ( unsigned long  ulParamID,
char *  pszParam 
)

Set the initial parameters.

Parameters
ulParamIDSpecifies the type of the parameters. It can be set to one of the following:
SDSC_DEVICE_PATH_NAME: Specifies the device file name and the mountpoint of the device in order, the device file name and the mountpoint must be separated by ','.
All devices can be input by multi-string.
It means there are two '/0' in the trail. Before calling SDSCListDevs, the calling application can call SDSCInitParams to set SDSC_DEVICE_PATH_NAME.
For example, if there are two devices: "/dev/sdb1,/media/disk", "/dev/sdc1,/media/disk-1", the two strings can be input by multi-string.
If the calling application does not set SDSC_DEVICE_PATH_NAME, SDSCListDevs will search all devices in "/dev/".
pszParamPointer to a null-terminated string.
Returns
Returns 0 on success; otherwise, returns an error code.
